Express.js Full Notes 1. Introduction to Express.js Express.js is a minimalist, unopinionated web framework for Node.js. It provides a robust set of features to build web and mobile applications, including routing, middleware, templates, and more. 2. Setting Up Express.js 2.1 Installing Express To get started with Express, you first need to install it: bash # Initialize a new Node.js project npm init -y # Install Express npm install express 2.2 Basic Express Server Setup Here's how you can create a basic Express server: js const express = require('express'); const app = express(); const port = 3000; app.get('/', (req, res) => { res.send('Hello, World!'); }); app.listen(port, () => { console.log(`Server running at http://localhost:${port}`); }); In this example, Express is set up to respond with "Hello, World!" when the root URL is accessed. 3. Express Routing 3.1 Routing Basics Routes in Express define how the app responds to client requests for specific URLs and HTTP methods (GET, POST, etc.). js app.get('/', (req, res) => { res.send('GET request to the homepage'); }); app.post('/submit', (req, res) => { res.send('POST request to /submit'); }); 3.2 Route Parameters You can define dynamic routes with parameters: js app.get('/user/:id', (req, res) => { const userId = req.params.id; res.send(`User ID: ${userId}`); }); In this case, the :id is a route parameter that can be accessed using req.params. 3.3 Query Parameters You can also capture query parameters using req.query: js app.get('/search', (req, res) => { const query = req.query.q; res.send(`Searching for: ${query}`); }); 3.4 Multiple Routes You can chain multiple routes for the same path with different HTTP methods: js app.route('/user') .get((req, res) => res.send('GET request to /user')) .post((req, res) => res.send('POST request to /user')) .put((req, res) => res.send('PUT request to /user')); 4. Middleware in Express Middleware is a function that has access to the request, response, and the next middleware function in the applicationâ€™s request-response cycle. 4.1 Basic Middleware Middleware functions are used to process requests before they reach your route handlers: js app.use((req, res, next) => { console.log('Request received'); next(); // Pass control to the next middleware function }); 4.2 Built-in Middleware Express has several built-in middleware functions, such as express.static() for serving static files and express.json() for parsing incoming JSON bodies. js // Serve static files app.use(express.static('public')); // Parse JSON bodies app.use(express.json()); 4.3 Error Handling Middleware To handle errors in your application, use a special error-handling middleware. This middleware must have four arguments: err, req, res, next. js app.use((err, req, res, next) => { console.error(err.stack); res.status(500).send('Something broke!'); }); 5. Express Templates 6.1 Template Engines Express supports template engines like Pug, EJS, and Handlebars for rendering dynamic HTML pages. Pug Example: bash npm install pug js app.set('view engine', 'pug'); app.get('/', (req, res) => { res.render('index', { title: 'Express', message: 'Hello World' }); }); 6.2 Rendering Views Use the .render() method to render views with a template engine. js app.get('/profile', (req, res) => { res.render('profile', { user: req.user }); }); 7. Handling Forms in Express 7.1 Parsing Form Data To handle form submissions, you need to use middleware to parse the form data. URL-encoded Form Data: js app.use(express.urlencoded({ extended: true })); JSON Data: js app.use(express.json()); 7.2 Handling POST Requests Here's an example of handling a POST request with form data: js app.post('/login', (req, res) => { const { username, password } = req.body; res.send(`Logging in as ${username}`); }); 8. Connecting to Databases You can integrate MongoDB, MySQL, or PostgreSQL with Express. 8.1 MongoDB Example (Using Mongoose) Install Mongoose: bash npm install mongoose Basic MongoDB setup: js const mongoose = require('mongoose'); mongoose.connect('mongodb://localhost/mydatabase', { useNewUrlParser: true, useUnifiedTopology: true }); const Schema = mongoose.Schema; const UserSchema = new Schema({ name: String, age: Number, }); const User = mongoose.model('User', UserSchema); app.get('/users', async (req, res) => { const users = await User.find(); res.json(users); }); 8.2 SQL Databases Example (Using Sequelize) Install Sequelize: bash npm install sequelize mysql2 Sequelize setup: js const { Sequelize, DataTypes } = require('sequelize'); const sequelize = new Sequelize('mysql://user:password@localhost:3306/mydatabase'); const User = sequelize.define('User', { name: DataTypes.STRING, age: DataTypes.INTEGER, }); app.get('/users', async (req, res) => { const users = await User.findAll(); res.json(users); }); 9.
